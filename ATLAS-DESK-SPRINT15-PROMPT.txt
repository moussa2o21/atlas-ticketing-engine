Incident aggregate rules:
  → TicketNumber generated on creation: "INC-{YYYY}-{NNNNN}" zero-padded sequential per tenant
  → Priority = f(Impact, Urgency) using standard ITIL priority matrix
  → State machine: only valid transitions allowed (guard conditions as private methods)
  → Each transition raises the corresponding domain event
  → IsMajorIncident = false by default; set only via DeclareMajorIncident() method
  → ResolutionNotes required when calling Resolve()

DeskDbContext rules:
  → Schema resolved from ITenantContext.SchemaName at runtime
  → Override SaveChangesAsync to set CreatedAt, UpdatedAt, CreatedBy, UpdatedBy
  → Global query filter: .HasQueryFilter(e => !e.IsDeleted && e.TenantId == tenantContext.TenantId)
  → audit_log table: no EF tracking — raw SQL INSERT only, never UPDATE or DELETE

audit_log table schema (raw SQL in schema-migration-job.yaml):
  CREATE TABLE IF NOT EXISTS {schema}.audit_log (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL,
    entity_type VARCHAR(100) NOT NULL,
    entity_id UUID NOT NULL,
    action VARCHAR(50) NOT NULL,
    old_value JSONB,
    new_value JSONB,
    actor_id UUID,
    occurred_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    is_ai_action BOOLEAN DEFAULT FALSE,
    ai_confidence FLOAT
  );
  -- No UPDATE or DELETE permissions on audit_log — enforced via RLS
  CREATE POLICY audit_log_insert_only ON {schema}.audit_log FOR INSERT WITH CHECK (true);
  ALTER TABLE {schema}.audit_log ENABLE ROW LEVEL SECURITY;

Indexes to create:
  incidents: (tenant_id, status), (tenant_id, assignee_id), (tenant_id, created_at DESC)
  sla_timers: (tenant_id, status, resolution_due_at) — for Hangfire SLA job
  audit_log: (tenant_id, entity_id, occurred_at DESC)

SlaEvaluationJob logic:
  1. Query all active SLA timers where resolution_due_at < NOW() + 20min (warning window)
  2. For timers at 80%+ elapsed → publish desk.sla.warning to Event Bus
  3. For timers where resolution_due_at < NOW() → mark Breached, publish desk.sla.breached
  4. Job must be idempotent — safe to run multiple times

CreateIncident handler must:
  → Resolve TenantId from ITenantContext (never from command body)
  → Generate TicketNumber via IIncidentRepository.GetNextTicketSequenceAsync
  → Calculate Priority from Impact × Urgency matrix
  → Create SLA timer immediately using matching SLA policy (Critical: 1h/4h, High: 4h/8h, Medium: 8h/24h, Low: 24h/72h)
  → Raise IncidentCreatedEvent
  → Write to audit_log
